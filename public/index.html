Danganengine

This demo should work fine in any modern navigator, but it's not guaranteed since it's just a pre alpha. The sprites, the courtroom, the game HUD and the camera movements are customizable, and the whole script is read in realtime from a JSON file.

I'll go to the point. The objective for this is offer a module package to be able to make more than just that demo. With the right code, it would be easy make a chat application to RP class trials in a near-game environment, or program a story in game format.

I started working in this 2 years ago, but only for a few weeks a year. A month ago I decided to take it more seriously and apply some things I've learned in these 2 years, node.js and modules specifically. As I said earlier, the project is currently in prealpha so the code will change. The project is hosted on github if you want to follow the development. I'm still an amateur, so if anyone want to report bugs or send pull requests, you're welcome to.

Distribution

Most of the resources to make this demo work are copyright from the game's developer, so I can't distribute them in this repo. However, if you can get them from other sources, this is the tree structure that makes it work:

  • /public
    • /resources
      • /fonts
        • aero.ttf: It's a free font available from dafonts.com
        • goodbyedespair.ttf: A fan-made ripoff based on the pixel font used in the japanese version of Super Danganronpa 2.
      • /sprites
        • /<character first name>
          • 1.png
          • 2.png
          • ...
        • /...
      • /textures
        • *.svg: I made these and are part of this repo.
        • alfombraF.png: plain red rug.
        • alfombraR.png: red rug with golden borders.
        • chapter1-guardapolvo.png: wood baseboard.
        • chapter1-marcos.png: golden pillars on the sides of the curtains.
        • chapter1-salida.png: exit doors' curtains.
        • chapter1-wallpaper.png: blue wallpaper.
        • clouds.jpg: a 256x256 Photoshop generated image using the Clouds filter. Is used in the cyan bar.
        • wood.jpg: a 512x512 image texture of wood. Is used in the stands.

The sprites and textures from the game are available in tumblr and/or some game decompilation databases. You can use the browser inspector to check the files as they're requested.

Building

The project is being developed with ES6 javascript, in hopes to port it to TypeScript in the future.

If you want to make modifications to the source, you have to re-build it using gulp. You must have nodejs installed.

  1. Open a terminal in the root folder of the project.
  2. Execute npm install to get the required node_modules.
  3. Execute gulp pack to rebuild the discussion example.

In case gulp is not working, check if gulp-cli is correctly installed in you global environment.

You can check the gulpfile.js in the root folder for other methods to build other examples.
